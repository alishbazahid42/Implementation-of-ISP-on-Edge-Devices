WHAT IT DOES — SYENet ISP Slim Model
======================================

OVERVIEW
--------
SYENet (Simple Yet Effective Network) is a lightweight neural network for Image Signal
Processing (ISP). It converts RAW Bayer sensor data directly into clean RGB images,
replacing the traditional hand-crafted ISP pipeline inside cameras.

This archive contains:
  - The pretrained slim model (5,640 parameters, 26.6 KB)
  - Python benchmarking tools (PyTorch GPU + TFLite CPU)
  - A standalone CUDA C program that runs the model without Python or PyTorch
  - Full benchmark results across 2,417 test images

THE PROBLEM IT SOLVES
----------------------
When you take a photo, the camera sensor records light as a RAW Bayer-pattern image — a
mosaic of R, G, G, B pixels. Converting this to a proper RGB photo normally needs a
complex ISP pipeline: demosaicing, denoising, white balance, tone mapping, colour
correction, etc. This pipeline has dozens of hand-tuned parameters.

SYENet replaces this entire pipeline with one small neural network trained end-to-end
on real (RAW, Fujifilm) image pairs. It learns the mapping automatically.

NON-TECHNICAL SUMMARY
----------------------
Think of it like auto-correct for camera photos — instead of engineers writing rules
for how to fix each type of image problem, the network learns by example from thousands
of pairs of raw vs. good photos. The slim version is so small (26 KB) that it can run
on a cheap embedded chip, not just a powerful GPU.

NETWORK ARCHITECTURE (Slim / Re-parameterized)
-----------------------------------------------
Input:  (1, 4, 128, 128)  — 4-channel RGGB after Bayer unpacking, normalised [0,1]
Output: (1, 3, 256, 256)  — 3-channel RGB after PixelShuffle x2 upscaling

  HEAD  — QuadraticConnectionUnit (Slim)
            branch1 = Conv5x5(4→12) + PReLU + Conv3x3(12→12)
            branch2 = Conv5x5(4→12)
            output  = branch1 * branch2 + bias(12)
            Learns multiplicative interactions between the 4 Bayer channels.

  BODY  — QuadraticConnectionUnit (Slim)
            branch1 = Conv3x3(12→12)
            branch2 = Conv1x1(12→12)
            output  = branch1 * branch2 + bias(12)
            Refines features with spatial × channel-wise multiplication.

  ATT   — Squeeze-and-Excitation attention
            GlobalAvgPool → FC(12→12) → PReLU → FC(12→12) → Sigmoid → scale
            Recalibrates channel importance dynamically per image.

  TAIL  — PixelShuffle(×2) → Conv3x3(3→3)
            Upscales from 128×128 back to 256×256 full resolution.
            Pixel shuffle rearranges 12 channels into 3-channel × 2× spatial.

STRUCTURAL RE-PARAMETERIZATION
--------------------------------
During training, each convolution block has multiple parallel branches
(3×3, 1×3, 3×1, 1×1, depthwise, BN-only) to improve gradient flow and accuracy.
At inference, all branches are mathematically fused into a single equivalent
convolution — same output, fraction of the compute. This "slim" conversion is lossless:

  PSNR before slim conversion: 24.8341 dB
  PSNR after  slim conversion: 24.8341 dB  (identical)

  Parameters: 110,984 → 5,640  (19.7× fewer)
  Model size: 524.9 KB → 26.6 KB

QUADRATIC CONNECTION UNIT
--------------------------
Standard networks use additive skip connections: output = f(x) + x
SYENet uses multiplicative: output = branch1(x) * branch2(x) + bias

This creates a quadratic non-linearity that better captures the non-linear
RAW-to-RGB mapping without adding extra activation functions.

WHAT THE NUMBERS MEAN
----------------------
PSNR 24.83 dB:  A common quality threshold for ISP benchmarks is ~25 dB. This model
                is just under but competitive for its tiny size (26 KB vs typical
                ISP networks that are 1-100 MB).

SSIM 0.8556:    On a scale of 0-1, above 0.85 is considered high structural similarity.
                The model correctly reconstructs textures, edges, and colour gradients.

1.6 ms / 641 FPS (CUDA C): Fast enough to process video at 60 FPS with headroom,
                running purely on GPU with zero Python overhead.

USEFUL FOR THE COMMON MAN?
---------------------------
YES, in indirect ways:
  - This type of model already runs inside smartphone cameras (though as a larger,
    commercial version). Companies like Qualcomm, MediaTek, Apple use similar networks.
  - The slim version is small enough to fit on a microcontroller-class device
    (26 KB is smaller than most font files), enabling cheap camera modules to produce
    professional-quality photos.
  - For researchers or engineers: it demonstrates that reparameterization can make a
    19× smaller model with zero quality loss — a technique applicable to many domains.
